# ChatGPT画像アーカイブアプリ 要件定義

## 0. 追加仕様（2026-06-06）

PCとスマホで保存内容を特別な手動操作なしに共有するため、任意機能としてSupabase同期を追加する。

- アプリ本体は引き続きPWAとして提供する
- ローカル保存先はIndexedDBのまま維持する
- Supabase同期を有効にした場合のみ外部通信を行う
- Supabase Authでログインした同一ユーザーの端末間で同期する
- Supabase Databaseにはグループ、プロンプト、タグ、メモ、ステータス、評価などのメタデータを保存する
- Supabase Storageには最大1024px / WebP / 品質0.8に変換した画像のみ保存する
- 元画像はIndexedDB、Supabase、ZIPのいずれにも保存しない
- Supabase bucketはprivateとし、RLS policyで本人のデータだけアクセス可能にする
- ZIPエクスポート／インポートは非常用バックアップとして継続する

## 1. システム概要

### 1-1. システム名（仮）
**ChatGPT Image Archive**

### 1-2. システムの目的
ChatGPTで生成した画像を、生成時に使用した以下の情報とあわせて保存・管理・検索・再利用できるアプリを作成する。

- 生成画像
- プロンプト
- 必要に応じてネガティブプロンプト
- 添付して参考にした画像
- タグ
- メモ
- ステータス
- お気に入り情報

特に、**1つのプロンプトから複数枚の画像を生成したケースを、1つのグループとして管理できること**を重視する。

## 2. 開発方針

### 2-1. 基本方針
本アプリは画像生成そのものを行うものではなく、**ChatGPTで生成した結果を保存・整理する管理アプリ**とする。

### 2-2. 保存方針

- データは基本的に**ローカル保存**とする
- 外部サーバーは使用しない
- API連携は行わない
- PCとAndroidの両方で利用できるようにする

### 2-3. 対応環境

- Windows PC の Chrome / Edge
- Android の Chrome
- PWAとして利用可能にする

### 2-4. 実装方針

- Webアプリとして開発
- PWA対応
- ローカルストレージは **IndexedDB** を使用
- バックアップ／移行用に **ZIP形式でエクスポート／インポート** を提供する

---

## 3. 想定利用者

- 主にアプリ作成者本人
- 個人利用を前提とする
- ログインや複数ユーザー対応は不要

## 4. 想定ユースケース

### 4-1. 基本ユースケース

1. ChatGPTで画像を生成する
2. 生成画像を保存する
3. 使用したプロンプトをコピーする
4. 必要に応じて参考画像も用意する
5. アプリに登録する
6. 後からキーワード・タグ・グループ単位で検索する
7. 過去のプロンプトや画像を再利用する

### 4-2. 複数画像生成時のユースケース

1つのプロンプトで複数枚生成した場合に、以下のように扱う。

- 1つのプロンプト = 1つの**プロンプトグループ**
- そのグループの中に複数の生成画像が属する

例：

```text
グループ名：白ボード付きポートレート
  - 画像1
  - 画像2
  - 画像3
  - 画像4
```

## 6. 機能要件

### 6-1. 登録機能

#### 6-1-1. プロンプトグループ登録

ユーザーは、1つのプロンプトに紐づく1つのグループを作成できる。

##### 登録項目

- グループタイトル
- プロンプト本文
- ネガティブプロンプト（任意）
- グループメモ（任意）
- カテゴリ（任意）
- タグ（複数可）
- 作成日（任意または自動）
- 登録日（自動）
- 参考画像（複数可）
- 生成画像（1枚以上）

#### 6-1-2. 生成画像登録

各プロンプトグループには、1枚以上の生成画像を登録できる。

##### 生成画像ごとの登録項目

- 画像ファイル
- 画像タイトル（任意）
- 画像メモ（任意）
- タグ（グループ共通とは別に個別タグも可）
- ステータス
- お気に入りフラグ
- 評価
- 画像サイズ
- ファイル形式
- 登録日

#### 6-1-3. 参考画像登録

各プロンプトグループに対して、参考画像を複数登録できる。

##### 参考画像ごとの登録項目

- 画像ファイル
- 種別
  - 人物参考
  - 衣装参考
  - 構図参考
  - 背景参考
  - その他
- メモ（任意）

#### 6-1-4. 登録方法

PCでは以下に対応する。

- 画像ファイル選択
- ドラッグ＆ドロップ
- クリップボード貼り付け（可能なら対応）

Androidでは以下に対応する。

- 画像ファイル選択
- フォトピッカーから選択

※ 共有メニューからの直接取り込みは将来拡張とする。MVPでは必須ではない。

## 7. 非機能要件

### 7-1. 対応デバイス

- PCブラウザで快適に利用できること
- Androidスマホでも操作しやすいこと

### 7-2. UI/UX

- スマホでも片手操作しやすいレイアウト
- 長文プロンプトのコピーが容易
- 画像の視認性が高い
- 操作は直感的であること
- 入力フォームは過度に複雑にしないこと

### 7-3. 性能

- 一覧表示でストレスなく閲覧できること
- サムネイルを使用して表示速度を確保すること
- 元画像は保存せず、登録時に最大1024pxのWebP画像へ変換して保存すること
- 一覧・詳細・バックアップ・将来の同期では、この変換後WebP画像を使用すること

### 7-4. データ保持

- IndexedDBを利用してローカル保存すること
- ブラウザデータ削除時に消える可能性があるため、バックアップ機能を重視すること

### 7-5. セキュリティ・プライバシー

- 外部サーバーにデータ送信しない
- オフラインでも閲覧可能
- 個人利用前提であること
- パスコードロックは将来拡張とする

## 12. この要件定義の重要ポイント

### 12-1. プロンプト単位で管理する

画像1枚ごとではなく、**1つのプロンプトから生まれた複数画像をまとめて管理する**。  
これにより、後から見返す時に「どのプロンプトで作ったか」が分かりやすい。

### 12-2. 参考画像も保存する

ChatGPTで添付画像を元に生成した場合、その参考画像も残しておくことで、後から再現性が高くなる。

### 12-3. ローカル保存を前提にする

外部サービスへの依存を減らし、コストや実装負荷を下げる。

### 12-4. Androidでも使える構成にする

ネイティブアプリではなくPWAとすることで、PC・Android両対応を実現する。

## 13. 要件定義の確定版サマリー

本アプリは、**ChatGPTで生成した画像を、プロンプト単位のグループで保存・管理するPWA型ローカルアーカイブアプリ**である。

- 1つのプロンプト = 1つのグループ
- 1グループに複数の生成画像を所属させる
- 参考画像も保存する
- タグ、メモ、カテゴリ、お気に入り、ステータスで整理する
- PCとAndroidで使える
- ローカル保存し、ZIPでバックアップ／復元できる
- 元画像は保存せず、最大1024px / WebP / 品質0.8の軽量画像として保存する
